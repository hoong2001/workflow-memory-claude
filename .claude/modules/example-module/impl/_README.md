# impl/ — post-change implementation records

After finishing a change, leave a record of "how it was actually done, what was hit, why it ended this way".
- Produced by: `/workspace-save-implementation` (which also lightweight-syncs `<name>-flow.md`).
- Naming: same name as the matching plan in `plans/`, e.g. `2026-06-23-add-login.md`.
- This layer lets future-you (and Claude) trace back "how exactly we solved it last time".
