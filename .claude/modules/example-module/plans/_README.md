# plans/ — pre-change plans

Before touching this module, drop a plan here first, then act.
- Produced by: `/workspace-task-brief` (fuzzy requirement — clarify, then persist) or `/workspace-grill-with-docs` (clear goal — grill the plan, then persist).
- Naming: `<date>-<slug>.md`, e.g. `2026-06-23-add-login.md`.
- Pairs with the same-named file in `impl/`: left = "how I plan to do it", right = "how it was actually done".
