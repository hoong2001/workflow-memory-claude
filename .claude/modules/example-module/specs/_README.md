# specs/ — requirement specs (you provide)

Drop requirement documents here. They accumulate over time — multiple specs per module is fine.
- Source: provided by you, or discussed into existence via `/workspace-spec-discuss`.
- These are the INPUT that drives the work; `/workspace-task-brief` output and `plans/` are what Claude derives from them.
- Naming: `spec-<nnn>-<slug>.md` (e.g. `spec-001-customer-import.md`) or keep the original filename.
